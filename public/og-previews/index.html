How this gets rendered for real

When you pick a direction, I'll:

  1. Render all 12 vertical cards as 1200×630 PNGs (headless-chrome screenshot of this same HTML).
  2. Upload the PNGs to the existing Backblaze bucket (f005.backblazeb2.com/file/PB-HubSpot/og/<slug>.png).
  3. Install react-helmet-async and add <Helmet> to VerticalPage.tsx so each page emits its own <title>, og:title, og:description, og:image, og:url, Twitter card tags, and canonical URL.
  4. Add server-side prerender for crawlers (via vercel.json headers or a lightweight prerender step) — most social crawlers don't run JS, so the tags need to ship in the initial HTML response. I'll verify with a Vercel Edge middleware or prerender approach in the implementation step.
